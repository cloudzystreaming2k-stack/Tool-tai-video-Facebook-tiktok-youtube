# AGENTS.md

## Project Overview

### Project Name

Social Media Downloader

### Description

Desktop application built with Python and PyQt6 that allows users to download videos and audio from multiple social media platforms.

Supported platforms:

* YouTube
* Facebook
* TikTok
* Instagram
* Threads

Main features:

* Download videos in multiple qualities
* Download audio (MP3)
* View download progress
* Manage download history
* Configure application settings

---

# Technology Stack

## Backend

* Python 3.12+
* yt-dlp
* FFmpeg
* threading

## Frontend

* PyQt6

## Data Storage

### JSON Files

The application does not use a database.

JSON files are used for:

* Download history
* Application settings

# Layer Responsibilities

## GUI Layer

Responsible for:

* User interaction
* Displaying video information
* Showing download progress
* Displaying download history
* Managing settings

GUI must not contain business logic.

---

## Core Layer

Responsible for:

* Platform-specific implementations
* Video metadata extraction
* Format retrieval
* Download execution

Each downloader must inherit from BaseDownloader.

Example:

```python
class YoutubeDownloader(BaseDownloader):
    pass
```

---

## Services Layer

### DownloadManager

Responsible for:

* Start download
* Pause download
* Resume download
* Cancel download
* Download queue management

### URLParser

Responsible for:

* Detecting platform from URL
* URL validation

Example:

```python
{
    "platform": "youtube",
    "video_id": "xxxx"
}
```

### FileManager

Responsible for:

* Create directories
* Handle file names
* Prevent duplicate names
* Manage download folders

### FFmpegService

Responsible for:

* Merge video and audio
* Convert audio to MP3
* Format conversion

### HistoryService

Responsible for:

* Save download history
* Load history
* Delete history
* Export history

History must be stored in JSON format.

# History Rules

Each successful download must append a new record to history.json.

Example:

```python
history.append(download_record)
```

History file must be created automatically if it does not exist.

Maximum history records:

```text
1000 records
```

When the limit is exceeded:

* Remove the oldest records first.
* Keep only the newest records.
